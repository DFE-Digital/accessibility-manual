Understand visual impairments

Visual impairments affect how people perceive and interact with digital content. Common conditions include:

  • colour blindness (difficulty distinguishing certain colours)
  • low vision (reduced visual acuity requiring magnification)
  • blindness (relying on screen readers)
  • eye conditions (for example, cataracts or glaucoma)

Main challenges

Include:

  • reading small or low-contrast text
  • distinguishing information conveyed by colour alone
  • navigating with keyboard or screen reader
  • reading text when zoomed or magnified
  • understanding content when text spacing is adjusted

Essential testing tools

Automated testing tools

Automated tools can help identify common accessibility issues quickly. While they can't catch everything, they're a good starting point for testing.

axe DevTools

A browser extension that provides automated accessibility testing. It checks for contrast issues, missing text alternatives, and semantic structure problems. Available for Chrome, Firefox, and Edge.

Use axe DevTools to test for accessibility issues.

WAVE

WAVE (Web Accessibility Evaluation Tool) provides visual feedback about accessibility issues on your page. It highlights problems directly on the page and offers detailed explanations of each issue.

Use WAVE to test for accessibility issues.

Lighthouse

Built into Chrome DevTools, Lighthouse provides quick accessibility audits with actionable insights. It's particularly useful for checking basic accessibility requirements and performance metrics.

Manual testing tools

Manual testing tools help you simulate different visual impairments and verify accessibility features. These tools are essential for understanding how users with visual impairments experience your service.

Colour contrast checkers

Tools like the WebAIM Contrast Checker and Colour Contrast Analyser help verify that text meets WCAG contrast requirements. They're essential for ensuring text remains readable for users with low vision or colour blindness.

Visual simulation tools

Chrome DevTools includes built-in vision deficiency emulation, allowing you to simulate different types of colour blindness. You can access this through the Rendering tab in DevTools. This helps you understand how users with various types of colour vision deficiency might experience your content.

Screen readers

Essential tools for testing with screen reader users. NVDA is free for Windows users, while VoiceOver comes built into macOS and iOS devices. TalkBack is available on Android devices. These tools help verify that your content is properly structured and announced to screen reader users.

Remember: While automated tools are helpful, they can only catch about 30% of accessibility issues. Always combine automated testing with manual testing and real user feedback.

Testing checklist

Visual testing

  • Verify text contrast meets WCAG 2.2 requirements (4.5:1 for normal text, 3:1 for large text)
  • Test with different vision deficiency simulations
  • Ensure information isn't conveyed by colour alone
  • Check focus indicators are visible (3:1 contrast)
  • Verify text remains readable when zoomed to 200%

Screen reader testing

  • Verify all content is announced correctly
  • Check heading structure and navigation
  • Test form labels and error messages
  • Verify dynamic content updates are announced
  • Test keyboard navigation and focus management

Responsive testing

  • Test text spacing adjustments (line height, letter spacing, word spacing)
  • Verify layout remains usable at 200% zoom
  • Check touch targets are at least 44x44 pixels
  • Test with different viewport sizes

Testing with personas

GOV.UK provides detailed accessibility personas to help you understand and test for different user needs. These personas are based on real user research and help teams empathize with users' experiences.

Physical impairments affect how people perceive and interact with digital content.

Common conditions include:

  • difficulty using a mouse or trackpad
  • difficulty using a keyboard
  • difficulty using a touch screen
  • difficulty using a voice control

Challenges for people with physical impairments

Physical impairments can make it difficult to:

  • read small or low-contrast text
  • understand information conveyed by .......alone
  • understand speech in noisy environments
  • understand content when......

Testing tools to identify barriers

Run automated tests

Automated tools can help identify common accessibility issues quickly. While they can't catch everything, they're a good starting point for testing.

You could install and use:

  • axe DevTools
  • WAVE
  • ARC Toolkit

Run manual tests

After automated tests, you should do manual tests next. Manual tests can identify more accessibility problems and can let you see how your product or website works for people with different physical impairments.

You could use:

  • Target size
  • Resize text

Run screen reader tests

You should also test with a screen reader, to understand how people who use one might navigate your product or website.

Testing checklist

You can use an accessibility testing checklist to help you find accessibility issues relating to physical impairments.

Testing with personas

Use the GOV.UK accessibility personas to help you understand and test for different user needs. Each persona is based on user research.

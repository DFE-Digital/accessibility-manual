Headings should be used to create a logical structure for your content. This helps users to understand the content and navigate it more easily. They are an important way for screen reader users to navigate content.

The H1 of the page should describe to users what they can expect the main theme of the following content to be.

Users often skip through headings in a list to find the content they are looking for, unique, descriptive headings make for a good user experience for screen reader users.

Examples of the issues we find

Below are some examples of common issues we find where headings are an issue and how to avoid them.

Skipped heading levels

Skipped heading levels can cause confusion for users, and from a semantic code perspective, is incorrect.

Ensure headings are used in a logical order, for example, H1 followed by H2, H3, H4 etc.

You can use a tool such as Headings Map to check the heading structure of your page.

Do

Use a logical heading structure which clearly shows the hierarchy of the page.

Don't

Don't skip heading levels, this is confusing and semantically incorrect.

Use semantic heading structure

Using the correct code to create headings is important for screen reader users. Screen readers use the code to navigate the page and understand the structure of the content.

Semantic HTML includes the use of H1 to H6 tags to create a logical structure for your content.

This is instead of styling text to look like a heading, which can be confusing for screen reader users.

You can use a tool such as Headings Map to check the heading structure of your page.

Do

Use correct HTML markup (H1, H2 etc) for headings.

Don't

Don't style paragraphs or other elements to look like headings.

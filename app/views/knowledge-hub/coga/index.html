COGA is a set of 8 guidelines for inclusive design. By applying them to the products and websites we build, content will be more accessible

Who created COGA

The guidelines have been drafted by the World Wide Web Consortium (W3C) and invited experts. W3C also created WCAG (Web Content Accessibility Guidelines)

WCAG is a set of guidelines that help teams to test and audit products and websites to see how accessible they are. COGA is a set of guidelines for designers to consider when creating products and websites

The guidelines

At their core, the 8 guidelines are just good design. They have huge benefits for people with cognitive impairments

Cognitive accessibility can often be overlooked, as it is not addressed as significantly as other disabilities within WCAG. Cognitive disabilities do not always have clear solutions like other disabilities. For example, low vision users can use magnification tools or screen readers, whereas finding a universal approach for cognitive disabilities is more complex

The guidelines provide advice on how to make content usable for people with cognitive and learning disabilities. By following them, we can ensure that our products and websites meet the needs of people who face challenges with thinking, learning, remembering, and problem-solving

Help users understand what things are and how to use them

When creating content and components, it should always be clear to users what the purpose of an item is and how to use it. Consistent, identifiable content and visuals reduce cognitive load and make interaction easier

  • style buttons and links consistently
  • ensure users are aware of what will happen when interacting
  • use commonly recognized icons, such as a phone icon instead of a headset, to support people with English as an additional language
  • provide clear and helpful hint text for inputs
  • place commonly used components in consistent locations and order

Help users find what they need

Navigation should be easy and intuitive. Users should quickly identify important information, find features, and access content with minimal effort

  • provide a clear layout with defined sections
  • use a clear heading structure
  • make the most important information easy to find
  • include a sitemap
  • use good information architecture and navigation
  • make help and search features easy to recognize
  • ensure content is searchable

Use clear and understandable content

Content should always be clear, understandable, and easy to follow. Use plain language and layouts that prevent users from feeling overwhelmed

  • write in plain language, using short sentences, and avoid jargon
  • include images that support the content
  • ensure videos are easy to follow and include captions and a transcript
  • avoid large pages of written content
  • use a consistent, easy-to-read font
  • make choices clear
  • format content as users expect to see it (for example, date of birth)
  • use colour to enhance readability and urgency

Help users avoid mistakes and know how to correct them

Ask users only for the information you need. If an error occurs, explain what went wrong and how to resolve it

  • help users avoid mistakes by using patterns like “check your answers”
  • show examples of potential failures or errors
  • be flexible in data validation, for example, allow spaces in inputs
  • provide a back link
  • use clear error messages that include the problem, cause, and recommendation
  • provide a summary of errors
  • offer alternative methods for resolving issues, like a phone number or helpdesk link

Help users focus

Users do not want to be distracted. Avoid cluttered layouts, looping animations, or auto-playing content that can break concentration

  • use visual hierarchy to emphasize important content
  • provide one thing per page
  • don’t distract users mid-task, for example, with ads or signups
  • provide clear navigation aids to help users reorient themselves

Ensure processes do not rely on memory

Users should not have to rely on memory to recall important information or complete tasks

  • re-provide information when required for a task
  • make important information visible, not hidden in tooltips
  • provide hint text
  • allow copy and paste for inputs

Provide help and support

Users want accessible, consistently available support. Provide multiple mechanisms for support to accommodate different needs

  • link support at the source
  • provide help within error messages
  • link to documentation
  • help users self-serve where possible
  • provide multiple support channels

Support adaptation and personalization

Users want to use their own settings and preferences while interacting with your product or website

  • support dark mode and high contrast mode
  • support font settings like text and line spacing
  • use responsive units (rem, em) and avoid px
  • support reduced motion

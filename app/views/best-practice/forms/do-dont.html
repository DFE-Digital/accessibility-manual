Things to avoid when designing forms.

Descriptive error messages

Do

Use descriptive error messages that explain what the problem is and how to fix it.

Don't

Don't use generic error messages that are ambiguous.

Formatting of values

Do

Help users understand the expected format of an input.

Don't

Don't use technical terminology the user might not understand.

Link errors to inputs

Do

Linking the error message to the input makes it easier to find.

Don't

Missing links make it harder to find the problem if there are multiple inputs on a page.

Database errors

Do

Show a friendly error which helps the user understand the problem and how to fix it.

Don't

Don't expose server or database errors which contain information which can be used by a hacker.

Error summary and field error messages

Do

Ensure the error message in the summary and the field match.

Don't

Don't have different messages or generic messages as it could confuse the user.

Optional fields

Do

Tell users if a field is optional using hint text.

Don't

Don't make it ambigous as users could enter "Not applicable" which would lead to data issues.
